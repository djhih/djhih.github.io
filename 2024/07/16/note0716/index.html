Codeforces Round 958 (Div. 2)

(rated)

timeline: 11/22/91

A. Split the Multiset

給 n, k，n 代表 multiset 一開始的數字，k 代表一次操作可以將一個數字分成幾個數字。
求最少操作數讓 multiset 變成 n 個 1。
example:
n=5, k=2
{5} => {1, 4} => {1, 1, 3} => {1, 1, 1, 2} => {1, 1, 1, 1, 1}
ans = 4

想法

拆 k-1 個 1 跟 n-(k-1) 就好

B. Make Majority

給長度為 n 的 01 字串，每次操作可以選連續同樣的 bits 轉為當前眾數 (0 or 1)。
求是否能將字串轉為 1。

想法

如果原本 1 的數量多過 0 則 yes
如果 0 多，將連續的 0 全部變成一個 0，然後比對 0 1 數量。1 多則 yes，否則 no。

C. Increasing Sequence with Fixed OR

給一個數字 n，目標是一構造一個數列，相鄰兩數之間 or 結果為 n，並且嚴格遞增。
求最長的目標數列。

想法

n 可以分成 1 bit 跟 0 bit，將 1 bit 由左到右依次替換為 0。
example: 111(2) => {011, 101, 110, 111}

賽後想法

發現可以直接以公式算出答案，解會是 1 的各數 + 1，不用實際去跑一次。

這題太晚想到可以把題目轉成，把哪一位數換成 0。
賽中想法是 : 如果嚴格遞增的情況，目標會希望讓每次增加的數最少。
但這個貪心的問題會變成，如果讓差值最小，可能會有 or 結果變差的情況，因為題目需要相鄰的數 or 起來保持相同，也就是當前差值最小接下來差值會被放大。
後來打掉重練，發現可以把題目看成那些位數轉 0，就發現一次只轉一個 0 會讓答案變成最佳解。

寫這題的時候不確定自己是不是腦袋在空轉，好像寫了很多數字卻沒有仔細思考其中的規律。

D. The Omnipotent Monster Killer

給 tree，每個 node 有一個數值，每次操作可以把某些數值變成 0。
操作的規則是 : 不能消除相鄰的 node 的值，但可以消除很多 node 的值。
example: a-b-c-d
可以選擇消除 ac or bd
求在所有數值變成 0 之前每次操作剩餘的數值的總和。

想法

先想到這樣最快是兩次消完，然後想甚麼時後不是兩次消完最好，也就是一次消除不是把相鄰的全部消掉最好的時候，考慮 dp，感覺像是背包問題，但需要考量每回合拿最大的值並不一定會是
最佳解。
