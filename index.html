Codeforces Round 958 (Div. 2)

(rated)

timeline: 11/22/91

A. Split the Multiset

給 n, k，n 代表 multiset 一開始的數字，k 代表一次操作可以將一個數字分成幾個數字。
求最少操作數讓 multiset 變成 n 個 1。
example:
n=5, k=2
{5} => {1, 4} => {1, 1, 3} => {1, 1, 1, 2} => {1, 1, 1, 1, 1}
ans = 4

想法

拆 k-1 個 1 跟 n-(k-1) 就好

B. Make Majority

給長度為 n 的 01 字串，每次操作可以選連續同樣的 bits 轉為當前眾數 (0 or 1)。
求是否能將字串轉為 1。

想法

如果原本 1 的數量多過 0 則 yes
如果 0 多，將連續的 0 全部變成一個 0，然後比對 0 1 數量。1 多則 yes，否則 no。

C. Increasing Sequence with Fixed OR

給一個數字 n，目標是一構造一個數列，相鄰兩數之間 or 結果為 n，並且嚴格遞增。
求最長的目標數列。

想法

n 可以分成 1 bit 跟 0 bit，將 1 bit 由左到右依次替換為 0。
example: 111(2) => {011, 101, 110, 111}

賽後想法

發現可以直接以公式算出答案，解會是 1 的各數 + 1，不用實際去跑一次。

這題太晚想到可以把題目轉成，把哪一位數換成 0。
賽中想法是 : 如果嚴格遞增的情況，目標會希望讓每次增加的數最少。
但這個貪心的問題會變成，如果讓差值最小，可能會有 or 結果變差的情況，因為題目需要相鄰的數 or 起來保持相同，也就是當前差值最小接下來差值會被放大。
後來打掉重練，發現可以把題目看成那些位數轉 0，就發現一次只轉一個 0 會讓答案變成最佳解。

寫這題的時候不確定自己是不是腦袋在空轉，好像寫了很多數字卻沒有仔細思考其中的規律。

D. The Omnipotent Monster Killer

給 tree，每個 node 有一個數值，每次操作可以把某些數值變成 0。
操作的規則是 : 不能消除相鄰的 node 的值，但可以消除很多 node 的值。
example: a-b-c-d
可以選擇消除 ac or bd
求在所有數值變成 0 之前每次操作剩餘的數值的總和。

想法

先想到這樣最快是兩次消完，然後想甚麼時後不是兩次消完最好，也就是一次消除不是把相鄰的全部消掉最好的時候，考慮 dp，發現就是簡單的背包問題。但賽中來不及實作，TBD。

A. Only Pluses

三個整數a、b和c，求最大 a×b×c。
最多進行5次以下操作：
選擇其中一個整數將其增加1。

看到這邊直覺是把最小的數增加就可以找到最佳解。
但沒時間證明，所以就直接唬爛過去了。

B. Angry Monk

給定 n, k，n 代表數字總和，k 代表 n 被分成幾個數字。
可以將 1, a 合併成 a+1，或是將 a 分成 a-1, 1。
求最少幾次可以將 k 個數合併成 n。

想法

看到之後想說直接把最大的留下，然後剩下的就直接全部分解，最後再一次合起來，因為合併只能合併 a, 1，也就是說兩個數字要合併其中一個一定要被拆成 1。但這是貪心，沒證明。

賽後證明

如果不是將最大的跟其他小的合併而是找一個小的跟其他合併，這樣代表需要把大的拆解，或是把合併出來的消除，如果把合併出來的消除就不是最佳解。所以做法會是選定一塊把其他都拆解掉，在這個解法下，如果不是選最大的留下就不會是最佳解。

官方想法

官解竟然是用公式，還真的沒想到。
我們可以將其中一個拆解成長度為1的片段，然後將這些片段連接到長度為y的燉鍋上。總共，我們將進行2x−1次操作。由於我們想連接k個片段，至少k−1個片段需要被拆解並連接到其他片段上。如果我們將某些片段連接到一個片段上，則拆解它們是沒有意義的，因為拆解這些片段時也需要移除這些已連接的片段。因此，我們會選擇一個片段，將所有其他片段連接到這個片段上。最優解是選擇最大尺寸的片段，並將所有其他片段連接到它上面。因此，答案是2⋅(n−mx)−k+1，其中mx是最大片段的長度。

實作

sort + diff

官解

C. Gorilla and Permutation

有兩個 function, f 和 g。
f(i) 代表在前 i 個數中，所有大於等於 k 的數字的和。
g(i) 代表在前 i 個數中，所有小於等於 m 的數字的和。
給定 n, m, k
n 代表數列由 1~n 組成並且每個數都洽出現一次。
求 f-g 的最大值。

想法

想到可以把 >k 的由大到小排，並放在前面，
把 <m 的由小到大排放到後在，這樣會讓 f 最大而 g 最小。
注意到的細節式 m=k 的時候應該把 m 放到最後面，因為 m 會是大於等於 k 的最小數並且是小於等於 m 的最大數，所以後到最後面可以保證 f 最大 g 最小。

官方想法

差不多

實作

2 個 pq

官解

短到讓人讚嘆

D. Test of Love

ErnKor願意為了Julen做任何事情，甚至游過充滿鱷魚的沼澤。我們決定測試這份愛。ErnKor必須游過一條寬1米，長度為n米的河。

這條河非常冷。因此，在整個游泳過程中（從0到n+1），ErnKor最多可以在水中游泳k米。為了人類的考量，我們在河中不僅加入了鱷魚，還有他可以跳上的原木。我們的測試如下：

最初，ErnKor在左岸，需要到達右岸。左岸和右岸分別位於0和n+1米。這條河可以用n段表示，每段長度為1米。每段包含原木’L’、鱷魚’C’或只是水’W’。ErnKor可以按以下方式移動：

如果他在表面上（即在岸上或原木上），他可以向前跳不超過m米（他可以跳到岸上、原木上或水中）。
如果他在水中，他只能游到下一段河段（或者如果他在第n米處，則游到岸上）。
ErnKor不能以任何方式落在有鱷魚的河段。
判斷ErnKor能否到達右岸。

這題的題目看很久沒有懂 swim 的規則，專注力 gone…

想法

其實這題我卡很久，到後面還是看了提示才想到的，賽中應該就想不出來了。
想法卡住的地方是 : 游泳距離的處理

主要是想靠 dp 去解，但 dp 的狀態設得不好，一直每想到要怎麼把游泳距離放到 dp 式子中。

然後想 greedy 的時候在煩惱如果現在游了之後卻發現可以直接跳過去的困擾。

這邊結合官方想法一起講。
dp 的部分應該是把狀態設為到 i 的最小游泳次數。
greedy 的部分則是，先跳到不能跳在討論要不要游，如果游了就要在看一次最遠能跳到哪邊，依此類推。

其實 greedy 的部分不難想，要再來回顧一下為甚麼 greedy 的部分會卡。
